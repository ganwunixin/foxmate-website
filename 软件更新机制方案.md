# FoxMate AI 软件更新机制方案

## 概述

本文档详细说明 FoxMate AI 桌面软件的两种更新方案：**基础版本检测**和**一键自动更新**。旨在为用户提供无缝的软件更新体验。

---

## 方案一：基础版本检测 + 提示下载

### 方案特点
- **实现简单**：开发周期短（约半天）
- **用户可控**：用户自主决定是否更新
- **风险低**：不会影响现有功能

### 技术架构

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   客户端     │    │   服务器    │    │   用户界面   │
│  FoxMate AI │───▶│ 版本信息API │───▶│  更新提示框  │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 实现步骤

#### 1. 服务器端 - 版本信息文件
**位置**：`https://foxmate.sfo3.cdn.digitaloceanspaces.com/version.json`

**内容示例**：
```json
{
  "version": "1.1.0",
  "release_date": "2026-03-15",
  "download_url": "https://foxmate.sfo3.cdn.digitaloceanspaces.com/FoxMate%20AI.zip",
  "changelog": "新增专注度统计功能，优化AI对话体验",
  "file_size": "420MB",
  "checksum": "md5:abc123def456..."
}
```

#### 2. 客户端 - 版本检测模块
**位置**：`launcher.py` 启动时检测

**检测逻辑**：
```python
# 启动时检查版本
def check_update():
    current_version = "1.0.0"  # 当前版本
    
    # 从服务器获取最新版本信息
    latest_info = requests.get("https://foxmate.sfo3.cdn.digitaloceanspaces.com/version.json")
    
    if latest_info["version"] > current_version:
        # 发现新版本，显示更新提示
        show_update_dialog(latest_info)
```

#### 3. 用户界面 - 更新提示框
**界面设计**：
```
┌─────────────────────────────────┐
│          🦊 发现新版本          │
├─────────────────────────────────┤
│ 版本：v1.1.0 (2026-03-15)       │
│                                 │
│ 📝 更新内容：                   │
│ • 新增专注度统计功能            │
│ • 优化AI对话体验               │
│ • 修复已知问题                 │
│                                 │
│ 文件大小：420MB                │
├─────────────────────────────────┤
│ [立即下载]    [稍后提醒]        │
└─────────────────────────────────┘
```

### 用户操作流程

```
用户打开 FoxMate AI
     ↓
软件自动检测版本
     ↓
发现新版本？
     ↓ 是
弹出更新提示框
     ↓
用户点击"立即下载"
     ↓
浏览器打开下载页面
     ↓
用户手动下载并安装
```

### 优点
- ✅ 开发快速，风险低
- ✅ 用户完全控制更新时机
- ✅ 不涉及复杂的安装逻辑

### 缺点
- ❌ 用户需要手动下载安装
- ❌ 更新体验不够流畅

---

## 方案二：一键自动更新

### 方案特点
- **用户体验好**：一键完成更新
- **自动化程度高**：后台自动处理
- **专业级体验**：类似主流软件更新

### 技术架构

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   客户端     │    │   服务器    │    │  更新程序   │    │   新版本    │
│  FoxMate AI │───▶│ 版本信息API │───▶│ Updater.exe │───▶│  软件文件   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 实现步骤

#### 1. 服务器端 - 增强版本信息
**内容示例**：
```json
{
  "version": "1.1.0",
  "release_date": "2026-03-15",
  "download_url": "https://foxmate.sfo3.cdn.digitaloceanspaces.com/FoxMate_AI_v1.1.0.zip",
  "changelog": "新增专注度统计功能，优化AI对话体验",
  "file_size": "420MB",
  "checksum": "md5:abc123def456...",
  "update_files": [
    {"file": "FoxMate AI.exe", "size": 158432112},
    {"file": "backend/run.py", "size": 24567},
    {"file": "AI Part/AI.py", "size": 18943}
  ]
}
```

#### 2. 客户端 - 智能更新模块
**核心功能**：
- **版本比对**：精确检测需要更新的文件
- **增量更新**：只下载变化的文件
- **断点续传**：网络中断后继续下载
- **完整性校验**：确保文件下载完整

#### 3. 独立更新程序
**设计思路**：
```
主程序 FoxMate AI.exe
     ↓
发现新版本，启动更新程序 Updater.exe
     ↓
主程序退出，释放文件占用
     ↓
更新程序下载并安装新版本
     ↓
更新完成，重新启动主程序
     ↓
更新程序自动退出
```

#### 4. 用户界面 - 智能更新对话框
**界面设计**：
```
┌─────────────────────────────────┐
│         🦊 软件更新中           │
├─────────────────────────────────┤
│ 正在下载新版本 v1.1.0...        │
│                                 │
│ ████████████████████ 85%       │
│ 已下载：357MB / 420MB          │
│ 速度：2.3MB/s                  │
│ 剩余时间：约 30秒              │
│                                 │
│ 📝 更新内容：                   │
│ • 新增专注度统计功能            │
│ • 优化AI对话体验               │
├─────────────────────────────────┤
│ [后台更新]    [暂停]            │
└─────────────────────────────────┘
```

### 用户操作流程

```
用户打开 FoxMate AI
     ↓
软件自动检测版本
     ↓
发现新版本？
     ↓ 是
弹出智能更新对话框
     ↓
用户点击"立即更新"
     ↓
后台自动下载新版本
     ↓
下载完成，自动安装
     ↓
软件自动重启
     ↓
更新完成，用户继续使用
```

### 高级功能

#### 1. 后台静默更新
- 用户不操作时自动下载更新包
- 下次启动时提示安装

#### 2. 版本回退机制
- 新版本有问题时可回退到旧版本
- 保留最近3个版本的备份

#### 3. 网络优化
- 多线程下载加速
- 智能选择最快的下载源
- 断点续传支持

### 优点
- ✅ 用户体验极佳
- ✅ 更新过程完全自动化
- ✅ 专业级软件体验
- ✅ 支持增量更新，节省流量

### 缺点
- ❌ 开发复杂度高（约1-2周）
- ❌ 需要处理文件占用、权限等复杂问题
- ❌ 测试工作量较大

## 推荐实施路线

### 第一阶段（立即实施）
**采用方案一：基础版本检测**
- 快速上线更新提醒功能
- 让用户知道有新版本可用
- 为后续优化奠定基础

### 第二阶段（用户量增长后）
**升级到方案二：一键自动更新**
- 当用户量达到一定规模时
- 投入资源优化用户体验
- 提供专业级更新体验

---

## 技术实现细节

### 版本号规范
采用语义化版本号：`主版本.次版本.修订版本`
- **主版本**：不兼容的API修改
- **次版本**：向下兼容的功能性新增
- **修订版本**：向下兼容的问题修正

### 安全考虑
- 使用HTTPS传输版本信息
- 文件完整性校验（MD5/SHA256）
- 数字签名验证（可选）

### 兼容性处理
- 支持Windows 10/11系统
- 处理用户权限问题
- 备份用户配置文件

---

## 总结

**建议采用分阶段实施策略**：

1. **立即实现方案一**，让用户能够及时获取更新信息
2. **观察用户反馈和下载数据**，评估更新频率
3. **当用户量增长后**，投入资源实现方案二
4. **持续优化更新体验**，提供最佳用户体验

通过这种渐进式方案，可以在保证产品质量的同时，合理控制开发成本和技术风险。
