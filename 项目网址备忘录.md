## 🔄 工作流程逻辑

### 网站更新逻辑
```
开发者修改代码
     ↓
本地测试通过 (http://localhost:3000)
     ↓
推送到 GitHub (git push)
     ↓
Vercel 自动检测到更新
     ↓
Vercel 重新构建并部署网站
     ↓
用户访问最新版本网站
```

**关键点**：
- GitHub 是"触发器"，推送代码触发部署
- Vercel 是"执行者"，自动完成部署工作
- 整个过程完全自动化，无需手动操作

### 软件发布逻辑
```
开发者修改软件功能
     ↓
本地测试通过
     ↓
使用 PyInstaller 重新打包
     ↓
生成 FoxMate_AI_vX.X.X.zip
     ↓
上传到 DigitalOcean Spaces
     ↓
更新网站下载链接 (lib/config.ts)
     ↓
推送到 GitHub 触发网站更新
     ↓
用户看到新版本下载链接
```

**版本命名规范**：
- `FoxMate_AI_v1.0.0.zip` - 主版本.次版本.修订版本
- 保留历史版本，便于用户回退

### 版本更新检测逻辑（未来）
```
用户打开 FoxMate AI 软件
     ↓
软件访问 Vercel API: https://foxmate-website.vercel.app/api/version
     ↓
获取最新版本信息
     ↓
与当前版本比较
     ↓
发现新版本？
     ↓ 是
弹出更新提示框
     ↓
用户点击"下载更新"
     ↓
打开 DigitalOcean 下载链接
     ↓
用户手动安装新版本
```

**API 设计**：
```json
{
  "version": "1.1.0",
  "download_url": "https://foxmate.sfo3.cdn.digitaloceanspaces.com/FoxMate_AI_v1.1.0.zip",
  "changelog": "新增功能说明",
  "file_size": "420MB"
}
```

## 🚨 紧急情况处理

### 网站无法访问
1. 检查 Vercel 控制台：`https://vercel.com/`
2. 查看部署状态是否正常
3. 检查 GitHub 代码是否有问题

### 下载链接失效
1. 登录 DigitalOcean：`https://cloud.digitalocean.com/`
2. 检查 Spaces 中的文件是否存在
3. 更新网站 config.ts 中的下载链接

### 代码推送失败
1. 检查网络连接
2. 确认 GitHub 账号权限
3. 使用批处理文件避免中文路径问题
