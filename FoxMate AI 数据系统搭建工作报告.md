## 一、项目概述

### 1.1 项目背景

FoxMate AI 当前版本已完成基础功能开发，但缺乏用户行为数据收集机制，无法获取真实使用数据用于模型优化。本项目旨在搭建完整的数据收集、存储、处理系统，为产品迭代提供数据支撑。

### 1.2 项目目标

**核心目标：** 构建完整的数据-模型-产品闭环

**具体目标：**
1. 建立用户行为数据自动收集系统
2. 搭建云端数据库存储用户数据
3. 开发数据导出和模型训练流程
4. 实现软件自动更新机制

### 1.3 数据收集流程

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户使用电脑                              │
│                    (浏览网页、编程、娱乐等)                        │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                      客户端数据收集模块                            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │ 窗口信息  │  │ 键盘活动  │  │ 鼠标活动  │  │ 专注度   │        │
│  │ 采集     │  │ 监测     │  │ 监测     │  │ 计算     │        │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘        │
│                         每分钟记录一次                            │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                        本地数据存储                               │
│              user_data/activity_YYYYMMDD.jsonl                  │
│                   (JSONL格式，每行一条记录)                        │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                      数据上传模块                                 │
│  ┌──────────────────────────────────────────────────┐          │
│  │  触发条件：每天一次 或 累积100条记录               │          │
│  └──────────────────────────────────────────────────┘          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                     │
│  │ 数据读取  │→│ 匿名化   │→│ 加密传输  │                     │
│  └──────────┘  └──────────┘  └──────────┘                     │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                      云端API接口                                  │
│                    POST /api/collect                            │
│  ┌──────────────────────────────────────────────────┐          │
│  │  数据验证 → 格式检查 → 批量处理                    │          │
│  └──────────────────────────────────────────────────┘          │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Vercel Postgres 数据库                         │
│  ┌──────────────────────────────────────────────────┐          │
│  │  数据表：user_activities                         │          │
│  │  字段：用户ID、时间、应用、键盘、鼠标、专注度      │          │
│  └──────────────────────────────────────────────────┘          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐                     │
│  │ 用户索引  │  │ 时间索引  │  │ 应用索引  │                     │
│  └──────────┘  └──────────┘  └──────────┘                     │
└─────────────────────────────────────────────────────────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                      数据使用场景                                 │
│  ┌──────────────────┐  ┌──────────────────┐                    │
│  │   数据导出        │  │   模型训练        │                    │
│  │ (最近30天数据)    │  │ (重新训练模型)    │                    │
│  └──────────────────┘  └──────────────────┘                    │
│  ┌──────────────────┐  ┌──────────────────┐                    │
│  │   数据分析        │  │   版本发布        │                    │
│  │ (用户行为分析)    │  │ (发布新版本)      │                    │
│  └──────────────────┘  └──────────────────┘                    │
└─────────────────────────────────────────────────────────────────┘
```

**流程说明：**

1. **数据采集**：客户端每分钟自动采集用户行为数据（窗口信息、键盘活动、鼠标活动、专注度）
2. **本地存储**：数据先存储在本地 JSONL 文件中，确保数据不丢失
3. **自动上传**：每天自动上传一次，或累积100条记录后上传
4. **数据匿名化**：上传前移除敏感信息，生成匿名用户ID
5. **云端存储**：数据通过API接口存储到 Vercel Postgres 数据库
6. **数据应用**：数据用于模型训练、用户行为分析、产品优化

## 二、技术架构

### 2.1 整体架构设计

系统采用三层架构设计：

**客户端层：**
- 数据收集模块：实时采集用户行为数据
- 数据上传模块：自动同步数据到云端
- 版本检测模块：自动检测软件更新

**服务端层：**
- 数据接收 API：接收客户端上传的数据
- 版本信息 API：提供版本查询服务
- 数据库服务：存储用户行为数据

**开发端层：**
- 数据导出工具：从数据库导出训练数据
- 模型训练环境：重新训练优化模型
- 版本打包工具：生成可发布版本

### 2.2 技术选型

| 技术领域 | 技术方案 | 选型理由 |
|---------|---------|---------|
| 数据库 | Vercel Postgres | 免费额度充足，无需运维，与现有架构无缝集成 |
| API框架 | Vercel API Routes | Serverless架构，自动扩展，按需付费 |
| 数据格式 | JSONL | 轻量级，易于解析，适合流式处理 |
| 传输协议 | HTTPS | 安全加密，符合数据安全要求 |

## 五、技术亮点

### 5.1 架构设计

**Serverless 架构：**
- 无需维护服务器，降低运维成本
- 自动扩展，应对流量波动
- 按需付费，降低运营成本

**数据安全：**
- 数据匿名化处理，保护用户隐私
- HTTPS 加密传输，防止数据泄露
- 访问控制，确保数据安全

### 5.2 性能优化

**数据采集优化：**
- 轻量级采集方案，CPU占用 < 5%
- 异步处理，不阻塞主线程
- 本地缓存，减少网络请求

**数据传输优化：**
- 批量上传，减少请求次数
- 数据压缩，降低传输量
- 断点续传，提高成功率

### 5.3 可靠性保障

**容错机制：**
- 本地缓存，网络异常时数据不丢失
- 失败重试，自动恢复上传
- 错误日志，便于问题排查

**监控告警：**
- 实时监控系统状态
- 异常情况自动告警
- 快速定位和解决问题
